AIエージェントプロンプト作成の卓越ガイド

「優れたプロンプトは、AIの能力を最大限に引き出すための鍵である」

1. プロンプト構造の原則

1.1 明確な役割定義

プロンプトの冒頭で、AIの役割を明確に定義します。専門家としての立場、持つべき知識、守るべき原則を具体的に記述します。

推奨例:

あなたは5年以上のReact開発経験を持つシニアフロントエンドエンジニアです。
モダンなWeb開発の最新プラクティスに精通し、パフォーマンス最適化、
アクセシビリティ、保守性を重視したコード設計が得意です。

1.2 明確な指示構造

AIに求める行動を段階的に、明確に指示します。複数のステップがある場合は、順序立てて番号付きリストで提示します。

推奨例:

1. ユーザーの提供するReactコンポーネントを分析する
2. パフォーマンス、可読性、保守性の観点から問題点を特定する
3. 具体的な改善提案を行う
4. 改善後のコードを提示する
5. なぜその変更が効果的かを説明する

1.3 明確な出力形式

期待する出力の形式を具体的に指定します。見出し、箇条書き、コードブロックなど、構造化された出力形式を指定することで、一貫性のある結果を得られます。

推奨例:

以下の形式でレビューと改善案を提示してください：

## 1. コード分析
- 問題点A
- 問題点B
- 問題点C

## 2. 改善提案
```jsx
// 改善後のコード例
```

## 3. 説明
- 改善Aの理由と効果
- 改善Bの理由と効果

2. 専門性を高めるためのテクニック

2.1 暗黙知の明示化

AIに専門家として持っていて欲しい「暗黙知」を明示的に伝えます。これにより、より深い専門的視点からの回答を引き出せます。

小規模事業者向け受発注システム要件定義書

1. プロジェクト概要

本プロジェクトは、漁業、水産業、林産業、加工業社、商社などの小規模事業者向けに、手頃な価格で利用できる受発注システムを開発し、月額サブスクリプションモデルで提供するものである。従来の高価な販売管理システム（弥生販売など）に代わる、使いやすく機能的なシステムを目指す。

2. 目標とゴール

  • ITリテラシーが低いユーザーでも直感的に使える操作性の実現
  • 月額1万円で提供できる適切な機能セットの実現
  • 受発注業務の効率化と適切な在庫・納期管理の実現
  • AI技術を活用した付加価値サービスの提供
  • 異なるユーザーロールに応じた適切な情報アクセス制御の実現
  • メーカー在庫の確保機能による効率的な在庫管理の実現

3. ターゲットユーザー

  • 主要業種: 漁業、水産業、林産業、加工業者、商社
  • 事業規模: 小規模事業者
  • ITリテラシー: 非常に低い（FAX主体の業務を行っているレベルを含む）
  • 利用シーン: 受発注業務、在庫管理、請求管理など日常の販売管理業務

4. 機能要件

4.1 マスタ管理機能

仕入先マスター

  • 仕入先基本情報管理（企業名、住所、連絡先等）
  • 契約条件管理（支払条件、納入条件等）
  • 取引履歴参照
  • API連携設定（在庫確認用）

得意先マスター

  • 得意先基本情報管理（企業名、住所、連絡先等）
  • 契約条件管理（請求条件、配送条件等）
  • 取引履歴参照
  • 与信管理
